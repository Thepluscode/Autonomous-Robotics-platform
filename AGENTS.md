# AGENTS.md

This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## Project

**Autonomous Ecosystem Architect** ŌĆö a dashboard for managing swarms of "ASI-controlled" drones and sensors that monitor rewilding zones (biodiversity, soil health, predator/prey balance). Two-tier app: FastAPI + MongoDB backend, React (CRA/Craco) frontend. Live PRD lives in `memory/PRD.md`.

## Run / Build / Test

### Backend (FastAPI, requires MongoDB on `localhost:27017`)
```bash
cd backend
pip install -r requirements.txt          # one-time
uvicorn server:app --host 0.0.0.0 --port 8001 --reload
```
- All routes mounted under `/api`. WebSocket: `ws://localhost:8001/ws/updates`.
- On first startup, the server creates the admin user from `ADMIN_EMAIL`/`ADMIN_PASSWORD` and writes credentials to `memory/test_credentials.md`. It also kicks off `simulate_drone_movements()` (background task that mutates patrolling drones every 5s).
- Required env (`backend/.env`): `MONGO_URL`, `DB_NAME`, `JWT_SECRET`, `ADMIN_EMAIL`, `ADMIN_PASSWORD`, `EMERGENT_LLM_KEY`. The server **refuses to boot** if `JWT_SECRET` is missing, shorter than 32 chars, or matches a known-default value (`fallback-secret-key`, `change-me`, etc.). Generate one with `python -c "import secrets; print(secrets.token_hex(32))"`. For local-only experiments, set `JWT_SECRET_ALLOW_INSECURE=1` to bypass the validator (every boot then logs a CRITICAL warning so this can't quietly survive into a deploy). `.env` is loaded with `override=False`, so explicitly-set environment variables (Railway, CI, your shell) win over the file. Optional: `DEV_MODE=1` makes the server write the admin password into `memory/test_credentials.md` on startup (off by default ŌĆö that file is a credential leak waiting to happen). CORS uses an explicit origin allowlist seeded from `FRONTEND_URL` + `CORS_ORIGINS` env vars with `allow_credentials=True` (`*` is incompatible with credentials, so cookie auth requires the named allowlist).

### Frontend (React 18 + Craco)
```bash
cd frontend
npm install                               # one-time
npm start                                 # craco start ŌåÆ http://localhost:3000
npm run build
npm test                                  # craco test (CRA Jest)
```
- `frontend/.env` must set `REACT_APP_BACKEND_URL` (defaults to `http://localhost:8001`). The frontend derives the WebSocket URL from this by swapping `http`ŌåÆ`ws`.
- Set `ENABLE_HEALTH_CHECK=true` to load the optional `plugins/health-check/*` modules (the dir is gitignored / not always present ŌĆö leave `false` unless you've added them).
- Path alias `@` ŌåÆ `frontend/src` (configured in `craco.config.js` and `jsconfig.json`).
- `@emergentbase/visual-edits/craco` is loaded only in dev and silently skipped if not installed.

### Tests
- `tests/` is the **pytest** suite (run from repo root: `pytest tests/`). Tests hit a live backend over HTTP ŌĆö they need MongoDB and the FastAPI server running. The whole module **auto-skips** when the backend isn't reachable, so a missing dev server doesn't masquerade as a failure. Override the target with `BACKEND_URL=http://host:port pytest tests/`.
- Run a single test: `pytest tests/test_api.py::test_no_id_leak_across_create_endpoints -v`. Add `-k <substring>` to filter, `-x` to stop on first failure.
- Shared fixtures live in `tests/conftest.py`: `api` (ApiClient that auto-prefixes `/api`), `admin_token` / `auth_headers` (logs in with `ADMIN_EMAIL`/`ADMIN_PASSWORD`, defaults to `admin@ecosystem.com`/`EcoAdmin2024!`), `seeded` (calls `POST /api/seed` once), `unique_name` (per-test UUID slug). Reuse these instead of re-implementing login/seed.
- `backend_test.py` was a legacy standalone CLI runner; **deleted**. All tests live in `tests/test_api.py` now.
- `test_reports/iteration_*.json` are previous run snapshots ŌĆö informational, not fixtures.

### Mongo `_id` leak ŌĆö fixed globally at import time
`motor.insert_one(doc)` mutates `doc` by attaching a non-serializable Mongo `_id`. Routes that returned that dict used to 500. Fixed once in `server.py` near the imports: a small monkey-patch of `AsyncIOMotorCollection.insert_one` pops `_id` from the input dict after insert. Every write site is now safe by default. The `insert_and_return` helper is preserved as a documented pattern (belt-and-suspenders) but is no longer load-bearing. Regression guard: `tests/test_api.py::test_no_id_leak_across_create_endpoints`.

### Drones route ŌĆö UUID regex enforces ordering safety
The literal `/drones/feeds` route used to depend on decorator ordering (it had to come before `/drones/{drone_id}` or FastAPI would match `feeds` as a drone_id). It now uses `Path(..., regex=_DRONE_ID_PATTERN)` (UUID-only) on `get_drone` / `update_drone` / `delete_drone`, so route order no longer matters. Non-UUID drone IDs return 422 (path validation). Regression guards: `tests/test_api.py::test_drones_feeds_not_swallowed_by_uuid_route` and `::test_drones_route_rejects_non_uuid_path`.

## Architecture

### Backend ŌĆö three modules in `backend/`
- **`server.py`** (~1570 lines) ŌĆö FastAPI app, all `@api_router` route handlers, auth helpers, `ConnectionManager`, startup/shutdown, CORS, motor `_id`-leak monkey-patch. New routes go here, in the existing grouping (auth ŌåÆ weather ŌåÆ interventions ŌåÆ forecasts ŌåÆ geofences ŌåÆ tasks/comments ŌåÆ reports ŌåÆ public ŌåÆ drones ŌåÆ zones ŌåÆ sensors ŌåÆ alerts ŌåÆ AI ŌåÆ dashboard ŌåÆ patrols ŌåÆ species ŌåÆ notifications ŌåÆ seed).
- **`models.py`** (~390 lines) ŌĆö every Pydantic request/response/entity model. Star-imported into server.py so handlers reference `Drone`, `Zone`, etc. by bare name. Add new models here.
- **`simulator.py`** (~110 lines) ŌĆö `tick_drone_simulation(db, manager)` and the supervised `run_drone_simulation_loop(db, manager)` plus the four `DRONE_*` constants. Db handle and ConnectionManager are passed in explicitly so this module has no compile-time dependency on server.py (no circular imports).

The User document is intentionally **not** modeled ŌĆö it's a raw Mongo doc keyed by `ObjectId`, with auth code converting on the way in/out. Don't try to "consolidate" it into models.py without rewriting `get_current_user`.

- **Mongo**: `motor.motor_asyncio` async client. Most documents use UUID `id` strings except `users`, which is keyed by Mongo `ObjectId` (auth code converts via `ObjectId(payload["sub"])`). Don't change this without updating `get_current_user`.
- **Auth**: JWT delivered via httpOnly `access_token` / `refresh_token` cookies (primary path used by the React app ŌĆö tokens never touch JS-readable storage). The login/register response bodies still echo the raw tokens so non-browser clients can use `Authorization: Bearer ŌĆ”`, but anything browser-side ignores them. Access tokens 60min, refresh tokens 7d. `require_role([...])` is the dependency for RBAC; `admin` always passes role checks (also enforced in the frontend `ProtectedRoute`). Currently only `/api/seed`, `/api/_internal/drone-tick`, and the `/ws/updates` WebSocket call `require_role` directly ŌĆö most CRUD routes are still wide open. The deliberate public surface is now declared in `PUBLIC_ROUTES` / `PUBLIC_ROUTE_PATTERNS` near the auth helpers (login flow, `/api/public/dashboard`, observations, attestation, `.well-known/keys.json`). The constants are read by the not-yet-enabled Phase-A middleware; flipping `AUTH_GATE_PHASE_A=1` in W2 will 401 anything outside the allowlist. When you add a new public route, edit `PUBLIC_ROUTES` and add a regression test in `tests/test_unit.py`.
- **Attestation lookback cap**: `GET /api/zones/{zone_id}/attestation?hours=N` clamps `N` to `ATTESTATION_MAX_HOURS = 168` (7 days). The previous 720-hour (30-day) cap let a single anonymous request bulk-export a month of signed observations. Auditors who legitimately need a longer window paginate.
- **Phase-A auth gate** (`AUTH_GATE_PHASE_A` env var): the `auth_gate_phase_a` middleware (added after CORS in `server.py`) reads `PUBLIC_ROUTES` / `PUBLIC_ROUTE_PATTERNS` and 401s anything outside the allowlist when the flag is `1` / `true` / `yes`. **Default-off** ŌĆö this commit ships the mechanism dark so reviewers can audit the data structure and rollback before the W2 flip. Flipping the flag back to unset / `0` is fully reversible. CORS preflights and the `/mcp/*` mount bypass the gate (the latter has its own `MCP_API_KEY` auth). WebSocket auth is unchanged ŌĆö the WS endpoint still does its own token check and closes with `4401`. Tests live in `tests/test_unit.py::test_phase_a_middleware_*` and exercise: flag-off pass-through, flag-on 401 on protected paths, public allowlist bypass, valid cookie/Bearer acceptance, refresh-token-used-as-access rejection, garbage token rejection, OPTIONS preflight bypass.
- **Public-surface lock** (`tests/test_unit.py::test_public_surface_is_locked`): walks `server.app.routes` and asserts every HTTP route falls into exactly one of three buckets ŌĆö public allowlist, gated (handler uses `Depends(get_current_user)` / `Depends(require_role(...))` OR calls them inside the body), or `W2_UNGATED_BACKLOG` (the explicit list of routes scheduled for W2 hardening). Adding a new ungated route fails the test with a hint pointing at the three remediation options. Hardening a backlog route is the W2 progress signal ŌĆö the test enforces removing it from `W2_UNGATED_BACKLOG` once it's gated. Detection covers both `Depends`-style auth and direct `get_current_user(request)` calls (used by `/api/auth/me`, `/api/missions/*` mutations, `/api/tasks/*`, `/api/comments` POST, etc.) via an AST walk of the handler source.
- **Roles**: `admin`, `field_operator`, `scientist`, `viewer` (`ROLE_PERMISSIONS` map). Page-level role gates live in `frontend/src/App.js`.
- **WebSocket**: `ConnectionManager` broadcasts to all sockets connected on `/ws/updates`. The endpoint accepts a JWT in three places, in this order: `?token=...` query param, `Authorization: Bearer <token>` header (for non-browser clients), or ŌĆö what the React client actually uses ŌĆö the `access_token` cookie that gets sent on the WS handshake. The frontend hook (`useWebSocket.js`) does **not** append `?token=...`; it relies entirely on the cookie. Invalid/missing/expired tokens close the connection with code 4401 (and the client stops reconnecting on that code). Clients send `"ping"` and expect `{"type":"pong"}`.
- **LLM**: imports `emergentintegrations.llm.chat` ŌĆö this repo ships a **local replacement** at `backend/emergentintegrations/llm/chat.py` that mirrors the proprietary SDK's interface (`LlmChat(api_key, session_id, system_message).with_model(provider, model)`, `await chat.send_message(UserMessage(text, image_urls))`). It dispatches to **real OpenAI** when `OPENAI_API_KEY` is set, **real Anthropic** when `ANTHROPIC_API_KEY` is set and `provider="anthropic"`, and otherwise produces a deterministic context-aware fallback (echoes a slice of the system message + prompt rather than a canned "Red Fox 92%" string). Optional env: `OPENAI_MODEL` (default `gpt-4o-mini`), `ANTHROPIC_MODEL` (default `claude-haiku-4-5`), `LLM_DISABLE=1` to force fallback. Note: `EMERGENT_LLM_KEY` (`sk-emergent-ŌĆ”`) in `.env` is rejected by api.openai.com directly ŌĆö use a real `OPENAI_API_KEY` for live AI. If you ever install the real `emergentintegrations` PyPI package, delete the local stub or it will shadow the install.
- **Background simulation**: `simulate_drone_movements()` runs `_tick_drone_simulation()` every `DRONE_TICK_INTERVAL_S` (5s). Each tick pulls drones with `status in {"patrolling","deployed"}`, looks up their assigned `zone_id`, and steps them toward the zone center by `DRONE_STEP_DEG` (Ōēł5km). Battery drains 0.05 + 5├Śdistance per tick down to a `DRONE_MIN_BATTERY` floor of 5. Position updates are then broadcast to all WebSocket clients as `{"type":"drone_positions","drones":[...],"ts":...}`. Tests can drive the simulator deterministically via the unlisted `POST /api/_internal/drone-tick` endpoint ŌĆö much faster than waiting for the loop. Drones with no assigned `zone_id` hover (no movement) instead of jittering.

### Frontend ŌĆö React 18 + Tailwind + Shadcn UI
- Entry: `src/index.js` ŌåÆ `src/App.js`. All routes are declared in `App.js`; protected routes wrap `DashboardLayout` (Sidebar + Header) and pass an optional `roles` array to `ProtectedRoute`.
- API layer: `src/lib/api.js` ŌĆö single axios instance configured with `withCredentials: true` so the httpOnly `access_token` / `refresh_token` cookies ride along automatically. There is no request interceptor that injects a Bearer header and no localStorage / sessionStorage token plumbing; the JS bundle never sees the raw tokens. The response interceptor handles 401 ŌåÆ `POST /api/auth/refresh` (which reads the `refresh_token` cookie server-side and rotates the `access_token` cookie) ŌåÆ retries the original request, with a single-flight queue so concurrent 401s don't trigger multiple refreshes. On refresh failure it hard-redirects to `/login`. Add new API groups here as named exports rather than calling axios directly from pages.
- Auth state: `src/contexts/AuthContext.js` (`useAuth()`, `<ProtectedRoute>`). On login/register the context only stores `{id, email, name, role}` in React state ŌĆö the response body still echoes raw tokens, but they are deliberately discarded because auth rides on the httpOnly cookies the backend set. Session restoration on reload calls `GET /api/auth/me` (which authenticates via the cookie); 401 triggers the api.js refresh path. `PUBLIC_PATHS = {"/login", "/register", "/public", "/gaia-prime"}` skip the auth fetch. `<ProtectedRoute>` does a hard `window.location.href = "/login"` redirect when there is no user, which also flushes any stale React state.
- WebSocket: `src/hooks/useWebSocket.js` auto-reconnects every 3s and pings every 25s.
- Pages: each top-level feature is one file in `src/pages/*.jsx`. UI primitives (Shadcn-style) live in `src/components/ui/`; only Sidebar/Header are in `src/components/layout/`.
- **`/gaia-prime` is the auditor surface, not a copy of `/public`**. `frontend/src/pages/GaiaPrime.jsx` is a calm, terminal-flavored landing that publishes the Ed25519 verification key (`/.well-known/keys.json`), live chain counters (`GET /api/public/provenance/stats` ŌĆö total signed observations, active zones, latest entry, source-type mix), per-zone signed-observation aggregate roots (`GET /api/zones/{id}/attestation`), and a curl cheatsheet for the verify endpoint (`POST /api/observations/verify`). It deliberately uses the regular Earthy theme, not the dark Gaia HUD on `/public` ŌĆö the audience here is "verify us" reviewers, not investors. **Counters are sourced from the same Mongo collection `/api/observations` reads**, so what the page shows and what an auditor pulls agree by construction (no hardcoded vanity numbers ŌĆö that would be greenwashing). The page only calls public endpoints (`PUBLIC_ROUTES`); don't introduce calls that would force auditors to authenticate. The `provenanceAPI` group in `src/lib/api.js` (`getPublicKey`, `getZoneAttestation`, `listObservations`, `getStats`) is the canonical wrapper.
- **State kit** (`src/components/state/` ŌĆö barrel-exported as `@/components/state`): `LoadingState`, `EmptyState`, `ErrorState`, `Skeleton`/`SkeletonRow`/`SkeletonCard`, and the global `Toaster`. **Every page must use these** instead of returning blank sections during loading or on failure ŌĆö empty surfaces are how silent failures masquerade as "no data". The Toaster mounts once in `App.js` and listens to the imperative store at `@/lib/toast`. Pages call `toast.success(title, { description })`, `toast.error(...)`, `toast.warning(...)`, `toast.info(...)` from anywhere ŌĆö there's no provider lookup or hook ceremony. We deliberately did not pull `sonner` / `react-hot-toast` for this; the pipeline is ~150 lines and shipping a third-party update on a security-prereq path was the wrong trade-off for "state-fidelity quarter".
- Maps use `react-leaflet` + `leaflet`. Charts use `recharts`. Don't add framer-motion/GSAP ŌĆö `design_guidelines.json` mandates calm 0.2s ease transitions only.

### Design system
`design_guidelines.json` is the source of truth for the visual theme (Earthy/Light, "Control Room" grid, moss-green primary, terracotta accent, Outfit/DM Sans/JetBrains Mono). Tailwind tokens are wired through CSS variables in `src/index.css`. Two non-obvious rules:
- All interactive elements need a `data-testid` in kebab-case.
- Do **not** use external placeholder image services; use only the URLs listed in `design_guidelines.json`.

## Working Conventions Specific to This Repo

- `test_result.md` is a structured agent-handoff file with a "DO NOT EDIT" header block. The main agent updates the YAML body; a separate testing agent reads it. Preserve the header verbatim if you touch this file.
- `memory/` holds the PRD and the auto-generated `test_credentials.md`. Don't commit additional secrets here.
- `gitignore.txt` (note: not `.gitignore`) is a reference file only ŌĆö this directory isn't a git repo. There's no CI; "deploys" don't exist locally.
- Default admin (per `backend/.env`): `admin@ecosystem.com` / `EcoAdmin2024!`. Useful for quick logins; rotate before exposing the backend.
- The mock `emergentintegrations` package means AI features always return the same canned strings ŌĆö when "the AI gave a suspiciously generic answer," that's why.
- `CLAUDE.md` (sibling file for Claude Code) is intentionally a near-duplicate of this file. When you change run/build/test commands, repo conventions, or the gotchas above, update both so the two agent harnesses stay in sync.
